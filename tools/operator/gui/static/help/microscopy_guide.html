Microscopy Ingest — how to add your images to gjesus3

This tool files your microscope images (AxioScan, Cell Observer, LSM 900) into the shared gjesus3 archive, reading the labels straight out of the image files. You pick a saved “recipe” for your instrument, point at your folder, and check the preview.

Before you start: this computer must reach the shared drive \\gjesus3\gjesus3\gjesus3-data. The top of the page should show it with a green OK.

Open the tool

Double-click the Microscopy Ingest shortcut. A small black window opens (leave it open — that's the engine) and your browser opens the page. If it doesn't, go to http://127.0.0.1:5000/.

The usual way: run a saved recipe

1

Pick your instrument and recipe

Choose your instrument (AxioScan 7, Cell Observer, or LSM 900), then pick the saved recipe for the kind of batch you're filing. The recipe already knows how to read your file names and where things go.
2

Point at your folder

Click Browse… and select the folder that holds this batch's images. Folders show up so you can confirm you picked the right place (files are greyed out — you choose the folder, not a file).
3

Fill in anything the recipe asks for

If the recipe leaves something blank (for example the researcher name or sample type), the page asks you for it. Required boxes are marked.
4

Preview and read the table

Press Preview. You get one row per image with its ID, sample, project and link name, plus how many are new vs already filed. Check it looks right.
5

Ingest

Leave “Dry-run” ticked and press Ingest for a safe rehearsal (writes nothing). When it looks right, untick Dry-run and press Ingest again to file for real. Re-running is safe — already-filed images are skipped.

Defining a new convention (less common)

If you have a new kind of batch with no recipe yet, the Builder lets you set up how the file names are read and where things go, preview it live on your real files, and Save it as a new recipe for next time. If you're not sure, ask the Data Office to set the recipe up with you the first time.

Troubleshooting

What you seeWhat to do
NAS box not green / “not a NAS root” Make sure it reads \\gjesus3\gjesus3\gjesus3-data and you're on the network.
Preview shows “0 new” Those images are already filed, or the folder/recipe don't match — double-check the folder and that you picked the right recipe for this batch.
A row shows a ⚠ warning Hover it to read the note. It usually means a value couldn't be read from the file; the image still files, just flag it to the Data Office if it looks wrong.
Golden rule: the Dry-run checkbox lets you rehearse with zero risk. When in doubt, dry-run first, then untick it.
