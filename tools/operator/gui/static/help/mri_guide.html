MRI Ingest — how to add your scans to gjesus3

This tool copies your MRI scans off the scanner and files them into the shared gjesus3 archive, with all the right labels filled in for you. You only make a few simple choices. It never changes anything on the scanner — it only reads.

Before you start, you need two things: (1) this computer can reach the shared drive \\gjesus3\gjesus3\gjesus3-data, and (2) the data office has set up the scanner password file on this machine. If either is missing, see Troubleshooting — it's a one-time setup the data office does.

Open the tool

Double-click the MRI Ingest shortcut. A small black window opens (leave it open — that's the engine) and your web browser opens the page. If the browser doesn't open, go to http://127.0.0.1:5000/mri yourself.

Step by step

1

Check the destination (top of the page)

The Destination NAS box should already say \\gjesus3\gjesus3\gjesus3-data with a green OK next to it. If it's not green, fix it before going on (see Troubleshooting).
2

List the studies on the scanner

Click “List studies on the scanner.” After a moment you'll see a list of your studies, newest first. Use the search box to narrow it down — type an animal number (e.g. m149) or a protocol code (e.g. 0618).
Studies with “phantom” in the name are calibration objects, not animals — those won't ingest (there's no animal to file them under). Pick real animal studies (their name has an animal number like m17 and a 4-digit protocol code).
3

Tick the studies you want, then Pull

Check the box next to each study, then click “Pull N selected.” The tool copies them down and shows the progress. When it finishes it automatically shows you a Preview (next step). You don't need to know or manage where the files go — they're cleaned up for you after.
4

Say who ran the scanner

Fill in Operator required — the person who ran the scan (usually you). This is the only box you must fill in.
5

Choose where the scans get linked (optional)

Destination project defaults to “per animal-protocol code” — each scan goes to its own project automatically. Leave it unless you specifically want them all in one named project, or no project at all.
6

Press Preview and read the table

Click Preview. You'll see one row per scan, with the ID, sample, project, and the link name. A line like “5 non-scan folder(s) skipped (normal ParaVision housekeeping)” is completely normal — those are the scanner's bookkeeping folders, not scans.
7

Ingest

Leave “Dry-run” ticked the first time and press Ingest — this is a safe rehearsal that writes nothing. When the table and log look right, untick Dry-run and press Ingest again to file the scans for real. Re-running is always safe: anything already filed is skipped.

Troubleshooting

What you seeWhat it means / what to do
The connection line says “No scanner credentials on this machine” The one-time password file isn't set up here. Ask the data office to add gjesus3_mri.cred to this machine. You can't pull without it.
The NAS box is not green / “not a NAS root” The destination isn't reachable. Make sure it reads \\gjesus3\gjesus3\gjesus3-data and that you're on the network. The tool won't let you pull until this is fixed (so you never wait on a pull for nothing).
Preview says “0 new scans” Either you picked a phantom/calibration study (no animal), or those scans are already in the archive. Pick a real animal study.
It says DICOMs were regenerated / placeholders made Normal. If the scanner export didn't include DICOMs, the tool fills them in where it can; otherwise it files an empty placeholder that the data office completes later. It never blocks.
Golden rule: this tool only ever reads and copies from the scanner — it can never change or delete anything there. And the Dry-run checkbox lets you rehearse with zero risk. When in doubt, dry-run first.
